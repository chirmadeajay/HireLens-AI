# Hackathon Submission Answers

## Project Name
HireLens AI - Resume to JD Screening Assistant

## Problem Description
Recruiters and hiring managers spend hours manually reading resumes, comparing them against a job description, identifying gaps, and preparing interview questions. This is time-consuming, repetitive, and inconsistent, especially when there are many applicants for one role.

## Target User / Profession / Domain
Recruiters, HR teams, startup founders, hiring managers, and career consultants.

## How the Solution Uses AI
HireLens AI uses AI to read unstructured resumes and a job description. It compares each resume with the role requirements and converts the analysis into structured output: fit score, match decision, strengths, missing skills, recruiter summary, interview questions, and risk flags.

## AI Tools / Platforms Used
- Groq API
- Llama 3.1 8B Instant model
- Python
- Streamlit
- PyPDF2
- python-docx
- Pandas

## How This Helps the User
A task that can take 2 to 5 hours manually can be reduced to a few minutes. The user gets a ranked shortlist, consistent candidate evaluation, interview questions, and downloadable reports. This saves time, reduces effort, and helps hiring teams make faster first-round screening decisions.
